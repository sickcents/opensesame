# Gemini queries structured geometry via tool calls, not vector embeddings

The facility's "vector understanding" is structured geometric data (polygons, points, real-world coordinates) in relational tables, not an embeddings/RAG store. Gemini answers spatial and status questions ("how far is the fire extinguisher from rack 3," "what's open in Room 204") through a small set of tool/function calls that query Postgres directly and return exact results — not similarity search. We considered embedding facility data into a vector database, but rejected it: relative-position and status questions are exact arithmetic and lookups, not fuzzy semantic retrieval, and RAG would add retrieval imprecision to reasoning that should be exact. Revisit if the system later needs to search unstructured content (manuals, incident notes) where semantic retrieval is the right tool.
